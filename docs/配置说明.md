# ⚙️ 配置说明

## 📌 通用规则

- **编码要求**: 所有配置文件强制使用 **UTF-8 编码**
- **变量支持**:
  - **内部变量**: `{变量名}`（如 `{guild_name}`）
  - **PlaceholderAPI 变量**: `%变量名%`（如 `%vault_eco_balance%`）

## 📁 目录结构


## GUI 配置详解

### 基础参数

| 节点 | 说明 | 取值范围/示例 |
|------|------|---------------|
| `row` | GUI 行数 | 1 ~ 6 |
| `title` | GUI 标题 | `'&e&l公会列表'` |
| `colored` | 是否解析颜色代码 | `true` / `false` |
| `use_papi` | 是否解析 PAPI 变量 | `true` / `false` |
| `hide_all_flags` | 是否隐藏物品 FLAG | `true` / `false` |

### 物品配置

| 节点 | 说明 | 示例 |
|------|------|------|
| `index` | 单物品位置 | `12`（1-54） |
| `indexes` | 多物品位置 | `"11-17,20-26"` |
| `material` | 物品材质 | `GOLD_INGOT` |
| `display_name` | 显示名称 | `'&a使用 &e{cost}个 &a金币'` |
| `lore` | 物品描述 | `['&f拥有：%vault_eco_balance%个', '&f公会名：{guild_name}']` |
| `skullTexture` | 头颅纹理 | （自定义） |
| `priority` | 优先级（仅动态 GUI） | 数字越大越优先 |

### 配置示例

```yaml
# 支付选择界面示例
row: 3
title: '&e&l请选择支付方式！'
items:
  money:
    index: 12
    icon:
      use_papi: true
      material: GOLD_INGOT
      display_name: '&a使用 &e{cost}个 &a金币'
      lores:
        - '&f拥有：%vault_eco_balance%个'
        - '&f公会名：{guild_name}'
        - '&c点击支付！'
```

## 商店配置详解

### 商品参数

| 节点 | 说明 | 示例值 |
|------|------|------|
| `name` | 商品名称 | `"公会升级"` |
| `reward_type` | 功能类型 | `GUILD_UPGRADE / GUILD_ICON / GUILD_TELEPORT` |
| `sell.price` | 售价 | `1000` |
| `sell.ConfirmGUI` | 确认界面配置 | （见默认配置） |

### 功能类型说明
- `GUILD_UPGRADE`: 升级公会最大人数
- `GUILD_ICON`: 购买公会图标
- `GUILD_TELEPORT`: 公会传送功能
- `GUILD_GOD`: 全员集结令
